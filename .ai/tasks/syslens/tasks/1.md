# Task 1: Fix UI Layout Issues

**Status:** Completed
**Started:** 2026-01-11
**Completed:** 2026-01-11

## Description

The UI has layout problems:
1. Pages no longer scroll (specifically the hardware tab)
2. Tab contents display below the CPU/memory/disk/network stat cards instead of in their proper position

## Current State (Before)

The `dock-container.component.ts` had `overflow: hidden` applied to the `.dock-main` div:
- Inline style: `style="grid-area: main; overflow: hidden; min-width: 0; min-height: 0;"`
- CSS class: `.dock-main { overflow: hidden; }`

This prevented the child content from scrolling, even though `app.component.ts` wraps router content in `<main class="h-full overflow-auto">`.

## Changes Made (After)

Changed `overflow: hidden` to `overflow: auto` in `dock-container.component.ts`:
- Line 40: Inline style now uses `overflow: auto`
- Line 134: CSS class now uses `overflow: auto`

This allows the main content area to scroll when content exceeds the viewport.

## Notes

- User reported issue indicating recent regression
- Root cause: `overflow: hidden` was added to `.dock-main` div in a recent change
- Fix: Change to `overflow: auto` to allow scrolling
